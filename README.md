# CardNav

![](docs/banner.webp)

发现卡网，先不用一页页翻。

CardNav 是一个围绕 AI 账号购买导航、卡网发现和商家合作展开的产品。它一方面把分散在不同商家里的商品信息整理成统一入口，让用户在真正点进某个站之前，先完成搜索、筛选、比较和初步判断；另一方面也给商家提供入驻、曝光、赞助和合作位置。

在线访问地址：[https://cardnav.xyz](https://cardnav.xyz)

CardNav 最开始的起点，其实是一个很具体、也很个人的购买困惑：我只是想更方便地买号，但一直搞不清楚这些东西之间到底是什么关系，也经常找不到号到底在哪里买。信息太杂，链接太碎，商品名字又不统一，想买一个账号，常常得自己翻很多页面。

尤其是当你真的想买 `ChatGPT Plus`、`Claude Pro`、`Gemini Pro`、`Grok`、`Cursor` 这类 AI 订阅、AI 账号或大模型账号时，问题往往不只是“在哪里买”，还包括：

- 哪些商家现在还有货
- 哪个价格更便宜
- 哪些标题其实不是同一种商品
- 哪些链接已经失效
- 哪些渠道是代订、成品号、共享号、卡密或 CDK

对很多国内用户来说，AI 账号购买早就不只是“去官网付费”这么简单。你可能会遇到海外银行卡、地区价格、代订、共享账号、卡密 CDK、成品号、渠道批发、Telegram 群、闲鱼、卡网这些完全不同的购买路径。便宜渠道确实存在，但它们通常分散在卡网、群聊、个人收藏夹和各种代购页面里，而且每个渠道卖的东西、擅长的品类、库存稳定性都不一样。

更现实的一点是，很多人并不是不想买官方订阅，而是确实买不起官网正价，或者觉得自己没有必要长期为最贵的官方方案买单。于是才会去找更便宜的 AI 账号购买渠道、更低价的 ChatGPT Plus、Claude Pro 或 Gemini Pro 订阅方案。

结果就是：用户想买一个 ChatGPT Plus、Claude Pro 或 Gemini Pro，可能要打开十几个网页，自己判断哪个有货、哪个最低、哪个商品名对应的到底是什么、哪个页面还有效。这个过程低效，也很容易买贵、买错，或者买到已经缺货的商品。

CardNav 想解决的，不只是用户找信息太低效，也包括商家缺少一个更直接的公开展示和合作入口。

它希望把这件事变得更简单一点：

- 先给你一个能直接搜索和浏览的公开入口
- 先把更值得优先看的结果排到前面
- 先让你知道大家最近都在搜什么、哪些商家更活跃
- 先帮你缩小范围，再决定要不要点进具体站点
- 允许用户直接提交网址，系统自动进入后续收录流程
- 也让商家可以提交入驻、获得曝光，甚至进一步和我们做合作

## 它做什么

- 聚合多个卡网商家与商品信息，提供统一浏览入口
- 支持按搜索词查找相关商家和商品
- 默认优先展示更值得先看的结果，降低手动翻页和逐站判断的成本
- 提供热门搜索词，帮助用户快速进入常见需求
- 提供提交商家入口，让新的站点自动进入后续收录流程
- 提供赞助商展示位与合作入口，帮助商家获取更多曝光
- 持续整理和更新商品、库存、价格与商家信息，让公开结果保持可用

它不卖货、不收款，也不替任何渠道做担保。它更像一个购买前的信息入口和价格雷达：帮用户少开几个网页，少踩一点信息差，也帮商家更高效地被看见。

## 适合谁

- 已经知道卡网是什么，但缺少一个统一入口来筛选商家的人
- 想先看看某类商品大概有哪些渠道在卖的人
- 不想自己维护一堆收藏夹、每次都重复打开多个站点的人
- 想快速发现近期更活跃商家的人
- 想让自己站点获得更多展示机会的商家
- 想通过赞助或合作触达目标用户的商家

## 产品逻辑

这个项目主要做三件事：

1. 持续收集、整理和更新商家、商品、价格、库存与搜索需求数据
2. 把这些数据组织成适合公开浏览的首页、搜索、提交收录、入驻和合作入口
3. 按站点活跃度、商品丰富度和相关性，把更值得先看的结果排在前面

所以它更像一个“AI 账号购买导航”和“卡网发现与合作入口”。用户看到的是更容易浏览的结果顺序、热门需求和清晰的跳转路径；商家看到的则是入驻、曝光和合作机会，而不是后台采集、刷新或评分细节。

## 本地运行

```bash
npm install
cp .env.example .env
npm run dev
```

默认读取当前目录下的 `.env`。

## 环境变量

```dotenv
DATABASE_URL=postgres://user:password@host:5432/cardnav
PORT=3000
SITE_URL=https://cardnav.xyz
```
