# Объяснение частых форматов выдачи аккаунтов

> Эта статья предоставлена **CardNav [cardnav.xyz](https://cardnav.xyz)**.

Хотя многие продавцы продают товары, которые все называются «аккаунтами», фактическое содержимое выдачи отличается. Некоторые — это данные аккаунта, которые можно напрямую использовать для входа в веб, а некоторые — JSON-конфигурации для релейных панелей, импортных инструментов или скриптов. Перед заказом подтвердите формат выдачи, чтобы уменьшить недопонимание и риски использования.

Сначала можно сделать базовое различение:

- **Прямой вход в веб и клиент**: в первую очередь выбирайте формат «логин/пароль», с которым можно напрямую войти на официальный сайт, или формат «четырехчастный RT» с паролем для прямого входа.
- **Импорт в инструменты и релейные сайты**: если выдача идет как session или разные JSON-конфигурации, они обычно больше подходят для импорта в сторонние релейные программы или панели управления.

## Логин и пароль
<!-- badge="Самый прямой" icon="key" -->

Продавец напрямую отправляет email и пароль, обычно можно напрямую войти через веб или клиент.

- Подходит тем, кто хочет напрямую войти в ChatGPT или клиент
- Не подходит тем, кто хочет массово импортировать в инструменты или конвертировать формат
- Легко спутать с другими «форматами аккаунта», поэтому все равно нужно подтвердить прямой вход

```txt
example@example.com | abc***XYZ
```

## Четырехчастный RT
<!-- badge="Прямой вход / конвертация" icon="key" -->

В одной строке одновременно есть аккаунт, пароль, `client_id` и `refresh_token`.

- Подходит тем, кто хочет и прямой вход, и возможную последующую конвертацию формата
- Не подходит тем, кто не понимает поля и хочет просто войти без разбора
- Название выглядит сложным, поэтому его легко ошибочно принять за формат только для импорта в инструменты

```txt
example@example.com----abc***XYZ----cli***123----rt_abc***xyz
```

## ChatGPT Session
<!-- badge="Session JSON" icon="api" -->

Информация сессии, полученная из уже вошедшей веб-страницы, не обычный логин и пароль.

- Подходит тем, кому нужен экспорт сессии и дальнейшая конвертация формата
- Не подходит тем, кто хочет напрямую вводить логин и пароль на странице входа
- Многие думают, что это равно аккаунту, но на самом деле нельзя использовать напрямую как логин и пароль

```json
{
  "user": {
    "id": "user_abc***xyz",
    "email": "example@example.com"
  },
  "account": {
    "id": "acct_abc***xyz",
    "planType": "plus"
  }
}
```

## Codex / CPA / Sub2API / Cockpit
<!-- badge="Формат импорта в инструмент" icon="toolbox" -->

Обычно это JSON для импорта в инструменты, панели или релейные программы.

- Подходит для импорта в локальные инструменты, массового управления и дальнейшей конвертации формата
- Не подходит тем, кто хочет только прямой веб-вход
- Когда в названии товара написано «аккаунт», именно этот формат новичкам легче всего купить по ошибке

```json
{
  "refresh_token": "rt_abc***xyz",
  "access_token": "eyJ***xyz",
  "email": "example@example.com"
}
```

## Как конвертировать

Если вы получили `логин/пароль` или `четырехчастный RT`, обычно сначала подтвердите, можно ли войти напрямую, а затем решайте, конвертировать ли в `Session` или другой формат импорта.

Если вы получили JSON вроде `Session`, `CPA`, `Sub2API`, `Cockpit`, обычно это для инструмента или релейной программы. Сначала смотрите инструкцию продавца, затем решайте, как импортировать.

## Подтверждение заказа

- **Проверка формата прямого входа**: внимательно подтвердите, относится ли содержимое card-code к формату выдачи вроде логина и пароля, который позволяет напрямую войти через веб.
- **Распознавание формата инструмента**: определите, являются ли данные продавца JSON-учетными данными для импорта в клиенты вроде Codex / CPA.
- **Проверка условий сервиса**: проверьте, указаны ли в деталях товара срок подписки, способ повторной проверки и время гарантии по риск-контролю.
