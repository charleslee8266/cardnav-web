# 自建中转站共享额度

> 本文由 **卡网大全 [cardnav.xyz](https://cardnav.xyz)** 提供。

如果你有一定技术能力，或者你和几个朋友都想用同一类高端模型，那么“购买账号 + 自建中转站 + 共享额度”会是一条很有吸引力的路径。它的逻辑，是把原本面向订阅用户的额度，转换成一套你们自己可控的 API 入口。

这条路的吸引力主要在成本。比如某些高额度订阅套餐，一个人可能根本用不完；如果几个人共享，平均下来会比单独走很多第三方路径更划算。

但它的门槛和风险也都更高。

个人或小规模共享场景里，常有人使用 `Sub2API` 这类项目。它的思路，是把订阅账号的可用能力转换成 API key 形式，方便你在本地工具里统一接入。

项目地址：

## 主要风险

### 协议风险

多人共享、订阅转 API、反向代理，本身就可能违反官方协议。

### 风控风险

低成本账号、共享行为、异常地区访问和大量调用，都会提高被风控的概率。

### 运维风险

你不仅要解决“能不能用”，还要负责服务器、代理程序、凭证管理和故障排查。

## 适合谁

- 有一定技术基础
- 能接受自己维护服务
- 目标是压低长期使用成本
